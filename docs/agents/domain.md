# Domain docs

This repository uses a single domain context.

## Before exploring

- Read `CONTEXT.md` at the repository root when it exists.
- Read ADRs under `docs/adr/` that touch the area being changed.
- If either is absent, proceed silently; create or update it only when domain-modeling work resolves an actual term or decision.

## Vocabulary

Use terms as defined in `CONTEXT.md` in ticket titles, code, tests, schemas, and documentation. If a needed concept is missing, treat that as a domain-modeling signal rather than inventing an unnoticed synonym.

## ADR conflicts

Surface any proposed change that contradicts an ADR. Do not silently override an accepted decision.
