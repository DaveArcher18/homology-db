# Materialize the named atlas Snapshot

## Question

Can one deterministic build materialize the family theorems, `RP^0..12`,
`CP^0..12`, `HP^0..2`, identities without duplicate Conceptual spaces, named
fixtures, bounded constructions, normalized Homology and completeness
assertions, representative qualified Models, and visible Model coverage into
one content-addressed Snapshot?

## Acceptance criteria

- Cite family formulas for all `RP^n` and `CP^n`; materialize immutable
  specializations for `0 <= n <= 12`; return no query-time specialization.
- Provide compact finite-simplicial-set Models for `RP^0..12` and independent
  abstract-simplicial checks for `RP^2..5`.
- Provide finite-simplicial-set Models for `CP^0..4`, independently check
  `CP^2` with the pinned 9-vertex triangulation, and record
  `model_not_qualified` for `CP^5..12`.
- Materialize `HP^0..2`, using the point and sphere identities for `HP^0` and
  `HP^1` and the pinned Sage triangulation for `HP^2`.
- Include the Poincare homology sphere, K3, Moore spaces, the thirteen pinned
  lens spaces, existing surfaces, tori, and spheres, plus one bounded wedge,
  product, suspension, smash, and mapping-cone or cofiber example.
- Record `RP^0 = CP^0 = HP^0 = *`, `RP^1 = S^1`, `CP^1 = S^2`,
  `HP^1 = S^4`, `RP^2 = N_1`, and `RP^3 = L(2,1)` as identity assertions
  without duplicating Conceptual spaces.
- Every assertion and Model obeys the source-pin and construction budgets;
  every missing Model remains visibly distinct from missing Homology.
