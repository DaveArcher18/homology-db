# Supersede the external-review gate

## Question

How must the append-only review record, onboarding guide, README, changelog,
and work state record that Gabriel Ong and Dan Isaksen must not be invited
until `named-atlas-review-v1` passes, without rewriting the earlier decision or
implying that `local-preview-60` changed?

## Answer

The public repository now leads with an explicit review hold. The retained
external-review procedure is visibly historical and says not to circulate the
60-subject preview as the proposed atlas. The append-only process adds a new
decision that supersedes only the timing/cohort recommendation of the earlier
decision, leaving every previous run and verdict intact.

`local-preview-60` remains unchanged and continues to treat missing `CP^2` as
a typed regression outcome. The proposed reviewer experience is the separate
`named-atlas-review-v1` candidate, whose gate requires grounded projective
families, the named corpus, production-like provenance, a qualified cross-
model slice, reviewed knowl-like definitions, and adversarial QA before an
invitation.

Gabriel Ong's informal product feedback is recorded independently from a
database review. A primary-source audit of David Roe's ICERM talk adds stable,
versioned, context-independent definition records and meaningful persistent
labels to the schema/API/QA tickets. Definition prose is explicitly not
assertion evidence. Inline dropdown rendering and mathematically encoded
graphics remain later UI work.
